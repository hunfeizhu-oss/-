# 07｜店铺视觉与旺铺装修（Alibaba Shop Visual System）

## 1. 技能定位
从经营定位、Slogan到旺铺首页模块，生成面向B端买家的视觉转化方案。

## 2. 适用场景
- 旺铺首页改版
- 需要设计PC/移动端Banner
- 需要把工厂实拍升级为高端B端视觉
- 需要明确店铺定位和Slogan

## 3. 必要输入
- 店铺链接/主营产品/目标市场
- 工厂优势、认证、产能、客户案例
- 现有图片素材或品牌色偏好

## 4. 标准输出
- 店铺经营定位
- 中英双语Slogan
- 旺铺11大模块结构
- Banner文案与画面提示词
- 首页装修执行清单

## 5. 执行流程
1. 诊断店铺定位、产品结构和买家信任缺口
2. 确定视觉主线：目标买家、核心产品、采购收益
3. 生成多版本Slogan和首屏Banner方案
4. 规划旺铺11大模块的内容与顺序
5. 输出可交付设计师或AI制图的提示词和版式说明

## 6. 推荐调用工具
`web_search`, `web_fetch`, `read`, `image_generate`, `image_edit`

> 工具名称为部署环境中的能力占位。若当前环境没有对应工具，应改用用户上传数据、公开网页检索或人工导出数据替代，不得虚构后台数据。

## 7. 质量标准
- 输出必须先给结论，再给依据，最后给动作。
- 所有数字、认证、价格、交期、客户信息必须来自用户资料或公开可核验信息。
- 缺失信息必须标注“未提供/未找到/需确认”，不得自行补全。
- 建议必须可执行，避免只写“优化”“加强跟进”等空泛表达。
- 涉及客户沟通时，优先给出可直接复制的英文话术，并附中文解释。

## 8. 合规与安全边界
- 不要求用户提供明文密码；优先使用已授权连接、平台导出文件或用户上传资料。
- 不绕开平台规则，不诱导违规引流或平台外交易。
- 不编造认证、合作客户、产能、价格、检测报告和付款信息。
- 自动发送、自动推送、批量采集类动作必须经过用户授权，并保留人工复核机制。
